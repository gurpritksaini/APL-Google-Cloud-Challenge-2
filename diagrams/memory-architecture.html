Embedding Strategy
ChromaDB uses sentence-transformers (all-MiniLM-L6-v2) as the default embedding function — runs locally, no extra API calls.
No Gemini Embedding API needed Runs in-process Semantic similarity search

Write Flow — After Each AI Turn
1
Gemini returns response
Backend receives raw text including ---GRASP_META--- block
↓
2
Parse metadata
Extract mastery_signals, teaching_strategy, current_concept from JSON block
↓
3
Store turn → conversation_turns
chroma_service.store_turn(session_id, role, content, meta)
↓
4
Mastery gate check
If mastery_signals == "solid" → proceed to step 5, else skip
↓
5
Store concept → concepts_mastered
chroma_service.store_concept(session_id, topic, concept, "solid", strategy)
Read Flow — Session Start
1
User selects topic
Frontend POSTs to /api/sessions with { topic }
↓
2
Query concepts_mastered
WHERE topic == selected_topic → returns list of prior mastered concepts
↓
3
Query sessions
Retrieve last known difficulty_level for this topic
↓
4
Build memory context string
memory_service.build_memory_context(topic) → formatted text block
↓
5
Inject into Gemini system prompt
"Learner previously mastered: closures, scope (intermediate). Build on this foundation."
